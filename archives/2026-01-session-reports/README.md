# 📋 Rapports de Sessions

Ce dossier contient les rapports de sessions de développement et de correction de bugs.

**⚠️ Note** : Ce dossier est ignoré par Git (fichiers temporaires de travail)

## 📁 Types de rapports

### Rapports de bugs
- **`BUG_FIX_*.md`** - Documentation des corrections de bugs
- **`DIAGNOSTIC_*.md`** - Analyses détaillées de problèmes

### Rapports de sessions
- **`RECAP_SESSION_*.md`** - Résumés des sessions de travail quotidiennes
- **`OPTIMISATION_*.md`** - Documentation des optimisations effectuées
- **`CORRECTIONS_*.md`** - Listes de corrections appliquées

## 📝 Organisation

Les rapports sont organisés par date au format `YYYY-MM-DD` dans le nom de fichier.

## 🔍 Utilisation

Ces rapports sont utiles pour :
- Tracer l'historique des modifications
- Comprendre les décisions techniques
- Retrouver les solutions aux problèmes rencontrés
- Former de nouveaux développeurs

## ⚙️ Archivage

Les rapports importants peuvent être déplacés vers `docs/dev/` pour archivage permanent.

Les rapports temporaires sont nettoyés périodiquement pour éviter l'encombrement.
